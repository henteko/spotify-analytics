## 📈 3つの主要指標

各レポートで評価される3つの指標:

### 指標1: 再生率（Playback Rate）
- **基準**: 80%以上が優秀、50%以下は要改善
- **評価ポイント**: リスナーがエピソードをどれだけ視聴したか
- **スコア配分**: 40点満点

### 指標2: 離脱ポイント（Dropout Points）
- **基準**: 離脱率20%以下が優秀、40%以上は問題
- **評価ポイント**: どの時点で、なぜリスナーが離脱したか
- **スコア配分**: 30点満点

### 指標3: エンゲージメント（Engagement）
- **基準**: フォロワーと再生数のバランス
- **評価ポイント**: フォロワーが実際にコンテンツを視聴しているか
- **スコア配分**: 30点満点

## 🎨 レポートの機能

### 1. ヘルスコアダッシュボード
- ゲージチャートで健康スコアを視覚化
- スコアの内訳（再生率/離脱率/エンゲージメント）

### 2. AI分析（Gemini 2.0 Flash）

- 成功要因の特定
- 改善エリアの指摘

- トピックごとの離脱率
- 高関心/低関心トピックの識別
- 視覚的なプログレスバー

### 4. タブナビゲーション
- **Playback Rate**: 時間帯別の視聴維持率
- **Dropout Analysis**: 離脱パターンとトピック分析
- **Engagement**: フォロワーと再生数の関係
- **Action Items**: 優先度別の改善提案

### 5. インタラクティブチャート
- Chart.js による動的グラフ

- プログレスバー（トピック別離脱率）

## 🎓 学習ポイント

### デモレポートから学べる重要なポイント:

1. **エピソードの長さの重要性**
   - 優秀: 40分（適切）
   - 改善必要: 60分（長すぎ）
   - 緊急: 30分（長さは問題ないが内容に問題）

2. **タイトルと内容の一致**
   - タイトルで約束した内容を必ず提供する
   - 「初心者向け」なら本当に初心者向けの内容にする
   - タイトルと内容のミスマッチは深刻な離脱を招く

3. **具体性の重要性**
   - 抽象的な話題（比較、将来展望）は離脱を招く
   - 具体的な実例、デモ、コード例が効果的
   - 「紹介・説明」型のコンテンツは離脱率が低い

4. **イントロと本編の連続性**
   - イントロで約束した内容を早めに提供する
   - イントロが良くても本編で離脱が起きる原因を理解する

5. **エンゲージメントの重要性**
   - フォロワーが増えても再生されないのは問題
   - エンゲージメント率 = 再生数 / フォロワー数
   - 既存フォロワーの期待を理解し、応える

## 💡 推奨する使い方

1. **まず優秀な例を確認**
   - `demo-excellent-report.html` を開く
   - 理想的な指標値を理解する
   - 成功要因を学ぶ

2. **改善が必要な例を分析**
   - `demo-improvement-needed-report.html` を開く
   - どのような問題が発生するか理解する
   - 改善アクションプランを確認

3. **緊急対応例から学ぶ**
   - `demo-critical-report.html` を開く
   - 深刻な問題の兆候を理解する
   - 避けるべきミスを学ぶ

4. **自分のエピソードと比較**
   - 実際のエピソードでレポートを生成
   - デモレポートと比較して現状を把握
   - 改善アクションを実行
